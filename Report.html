Introduction

Total knee/hip joint replacement (total knee/hip arthroplasty) is performed to restore function and relieve pain in patients with severely damaged knees. The surgery involves replacement of both the medial and lateral femorotibial joints and the patellofemoral joint. Although total joint replacement is an effective treatment, postoperative complications include blood clots, infection, and loosening or malalignment of the prosthetic component. Periprosthetic joint infection (PJI) is a serious complication occurring in 1% to 2% of primary arthroplasties, which is associated with high morbidity and need for complex interdisciplinary treatment strategies.

Dataset

This dataset is originally used for doctors to diagnosis whether the patient will get PJI or not. The dataset that used for PJI diagnosis contain 52159 samples, 67 features and 1 label.

Objective

This project aim to predict the PJI infection by given dataset. I have tried many differnet models to compare the performance of each. Besides, Also, since the data is used for medical application, it is important for us to make the model comprehensible.

Preprocessing

Remove low quality samples

Low quality samples that contain too many missing values will effect models’ performance. Therefore, I drop the samples that contain more than 25% features missing. However, because the labels in dataset is extremely imbalanced, if I simply drop the sample containing too much missing features, the Infected samples in remain dataset will be too less for model to learn. Therefore, I keep all the infected sample. The label distribtion of dataset before and after dropping is

Status Non-Infected Infected Total Sample
Before dropping 51280 879 52159
After dropping 27076 879 27955

Dataset Split

Split the dataset into train and test exclusive set with ratio 8:2. After the model is trained by train dataset, validate the model with validation set to estimate the model performance.

Undersampling

Since the dataset is extremely imbalance, the model wopuld tend to focus on the majority class without undersampling or oversampling. I’ve tried both undersampling and oversampling, and undersampling is much better. Therefore, I undersample the train dataset with ratio 0.1 and validation dataset remain the same. The distribution of train dataset will be:

Dataset Infected Infected(%) Non-Infected Total Sample
Train 718 9.09% 7180 7898

Notice that in both ANN and CNN model, I did not do any sampling method but use the sampler in pytorch. It can reach the same effect.

Feature Selection

In original dataset, there are total 66 features but not each of them is important. Therefore, I remove the feature whose variance is lower than 0.2, which is 36 features in total. The feature count before and after feature selection is:

Status Feature count
Before selection 66
After selection 30

Notice that in both ANN and CNN, I did not do any feature selection, because it is hard to decide whether the feature is important to neural network or not.

Missing Value Imputation

Some models can not handle the missing values in dataset. It is necessary for me to deal with the missing value first.

Numeric feature

For numeric missing value, I used IterativeImputer to deal with it. It will consider the samples’ others feature to fill in the missing one. It is the way better than imputing the missing value simply by median and mean or the feature.

Categoric feature

For cateforic missing value, I used SimpleImputer to deal with it. Although it will be better to consider other features in the sample, I think filled it with an additional class in enough. For all categoric missing values, I imputed them with 10.0.

Scaler

It is necessaty to scale numeric features to prevent the large number features dominant the prediction in specific models.

For SVM & LR & CNN & ANN

For these three model, large number feature will dominant the model. Therefore, I used stardard scaler to scale the numeric features.

For DT & RF

For these two model, large number features will not dominant the prediction. I’ve tried to scale numeric features by stardard scaler in these models and it will not effect the performance very much. Therefore, I didn’t do any scaler for these models.

One hot Encoding

For CNN & ANN, it is necessary to one hot encode categoric data because in NN, it is inappropriate to define multiple class in a input neuron.

Model

I’ve build a few kinds of model include:

  1. CNN
  2. ANN
  3. Logistic Regression(LR)
  4. Support Vector Machine(SVM)
  5. Decision Tree(DT)
  6. Random Forest(RF)

Since the target dataset is extremely imbalanced, most of the model’s class weight are tuned as {0:0.1,1:1}.

CNN(Convolutional Neural Network)

Model introduction

The model I built have been seperate into features extraction and classifing part. In feature extraction part, there are 2 convolutional layers with kernel size 1 and stride 1 and using Relu as activative function. After each convolutional layer, there is a 1 dimensional maxpooling layer with both kenel size and stride 1. In classifing part, there are 4 fully connected layers, and the ratio of each layer’s input and output dimension is about 4:1. After first 3 fully connected layer, I use Relu as its activative function and Softmax for the last one. You can see the image below for more information.

Results

Since the label in the original dataset is extremely imbalanced, accuracy in this case would not be a good way to estimate the model. Therefore, I used the addtional criteria below to compare the performance of each model.

Performance Metric Definition
Percentage Accuracy(ACC) (TP + TN) / (TP + TN + FP + FN)
Recall TP / (TP + FN)
Precision TP / (TP + FP)
F1-score \(\frac{2\times Recall\times Precision}{Recall+Precision}\)
Matthews Correlation Coefficient (MCC) \(\frac{TP\times TN - FP \times FN}{\sqrt{(TP+FP)\times(TP+FN)\times(TN+FP)\times(TN+FN)}}\)

Conclusion

As you can see the above, most of the model can identify the infected well with each Recall score higher than 0.6, which is impressive in such a imbalanced dataset. The most obvious different between models is Precision score that range from 0.05 to 0.90. It means that although some models can identify the infected well, many non-infected will be diagnosis as infected, which will cost many medical resource in practice. To my surprise, RF is the best model with the highest ACC, Precision, F1-score and MCC. It shows that although many innovative machine learning algorithm like ANN and CNN is designed these years, the classic one can still not be ignored.
